# Contribuire a Housekeep

## Prima di iniziare

1. Leggi [docs/developer/setup.md](docs/developer/setup.md).
2. Esegui `flutter analyze` e `flutter test test/` prima di aprire una PR.

## Pull request

- Descrivi il **cosa** e il **perché** in italiano o inglese, con frasi complete.
- Mantieni il diff focalizzato sulla feature o fix; evita refactor non richiesti.
- Se tocchi Hive, includi note su **compatibilità** (solo field in coda).

## Decisioni architetturali

Le scelte strutturali importanti sono tracciate in [docs/adr/](docs/adr/README.md). Per una nuova decisione significativa, aggiungi un ADR seguendo il template ivi descritto.

## Documentazione

Aggiorna `docs/` se introduci flussi nuovi (dati, build, UX sviluppatore).
